# Skill: validate-rag-demo

## Quando usar
- Em tarefas de ingest, retrieval, colecoes por tenant ou base documental da demo.

## Objetivo
- Validar se o RAG da demo esta coerente, tenant-aware e com falha segura.

## Checklist
1. Confirmar collection ou base usada por tenant.
2. Validar se a ausencia de base gera erro controlado.
3. Rodar perguntas controladas ligadas aos documentos ingeridos.
4. Registrar sinais de retrieval util, fallback ou ruido.

## Evidencias minimas
- tenant validado
- colecao/base validada
- perguntas usadas
- comportamento observado
