# Credenciais de Exemplo

Esta pasta existe para orientar formatos seguros de credenciais sem manter segredos reais no projeto.

## Regras
- Nao adicionar tokens, chaves privadas, client secrets ou `.env` reais.
- Use apenas exemplos anonimizados ou listas de campos esperados.
- Qualquer segredo encontrado em area ativa deve ser removido e rotacionado fora do repositorio.

## Contexto atual
- A estrutura antiga congelada em `.agent_legacy/credentials/` permanece apenas como referencia historica e deve passar por revisao manual antes de qualquer reaproveitamento.
